Plugin format

One manifest.
Every agent reads it.

SwarmSpace capabilities use a standardized OpenAPI-style manifest ‚Äî a JSON/YAML file that describes what your capability does, its trust tier, pricing, auth method, and semantic tags. Any LLM can parse and summarize it without reading full documentation.

Manifests are the contract between your capability and every agent in the SwarmSpace ecosystem. Write it once. Discoverable everywhere.

Agent-to-agent compatible. Each plugin has a stable canonical URL and machine-readable summary. Agents can pass plugin references to each other as structured objects ‚Äî inter-agent recommendation is a first-class use case.
oura-sync.swarmspace.json SwarmSpace Manifest v1

  "name": "OuraSync",
  "version": "1.2.0",
  "capability": "Read and analyze sleep, HRV,
    and readiness data from Oura Ring",

  "trust_tier": "verified",
  "latency_class": "realtime",
  "pricing": {
    "model": "free"

  "auth": {
    "method": "oauth2",
    "scopes": ["sleep", "heartrate"]

  "context_required": "none", // no PII needed

    "health", "sleep", "wearables", "HRV"

  "agent_summary": "Reads Oura Ring sleep

Trust architecture

Capabilities don't get to
decide how much they're trusted.

Every plugin sits in a trust tier. Every agent matches its permission level against the tier before execution. Users always know what they're authorizing.
